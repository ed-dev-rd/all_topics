Вариант 8. Табу-поиск для задачи планирования
-
Задача: реализовать табу-поиск для оптимизации расписания работников.

Требования:

- Каждый работник должен работать определенное количество дней в период
  
- Табу-лист для предотвращения циклического поиска
  
- Вывести финальное расписание и оценку качества
  
Входные данные:

5 работников, 7 дней, каждый должен работать 5 дней
Ограничения: не более 2 выходных подряд

Табу-поиск (Tabu Search)
-
Принцип: использует память о посещенных решениях для управления поиском.

Компоненты:

- Краткосрочная память (Tabu List) — список недавних ходов, запрещенных на текущей
итерации

- Долгосрочная память — информация о частотности и влиянии ходов

- Критерий аспирации — условие для перекрытия табу-статуса

Параметры:

- Размер табу-листа — обычно 7-20 элементов

- Количество итераций — критерий остановки

Временная сложность
-
В лучшем случае алгоритм находит решение за несколько итераций, но сложность все равно остается высокой из-за полного перебора всех возможных пар работников и дней при генерации соседей.

Формула:

W × D × W × D = W²×D² на одну итерацию.

Умножаем на количество итераций I: O(I×W²×D²)
